# Reflection

Version control is a system that tracks changes to files over time, allowing developers to collaborate without overwriting each other's work. I learned that Git records the complete history of modifications, while GitHub provides a platform to share and review that history with team members.

The most difficult step was understanding the branch and merge workflow, as it requires keeping track of multiple versions of the same project simultaneously and resolving conflicts when they arise.

Branches and pull requests are useful because they enable safe experimentation. You can develop features in isolation, review changes with team members, and merge only when everything is verified. This prevents bugs from entering the main codebase and supports collaborative development without conflicts.
